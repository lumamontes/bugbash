## Commands

- `pnpm dev` ‚Äî Start dev server at localhost:4321
- `pnpm build` ‚Äî Production build to `./dist/`
- `pnpm preview` ‚Äî Preview production build locally
- `pnpm astro check` ‚Äî TypeScript type checking
- `pnpm db:generate` ‚Äî Generate Drizzle migration SQL from schema changes
- `pnpm db:migrate` ‚Äî Apply pending migrations to the database
- `pnpm db:seed` ‚Äî Seed database with demo data (resets existing data)
- `pnpm db:studio` ‚Äî Open Drizzle Studio for database inspection

## Architecture

Astro 5 project with React integration, Tailwind CSS v4, and SSR via `@astrojs/node`.

- **Pages**: `src/pages/` ‚Äî file-based routing (`.astro` files)
- **Layouts**: `src/layouts/Layout.astro` ‚Äî base HTML layout; `src/layouts/app-layout.astro` ‚Äî sidebar layout for authenticated pages
- **Styles**: `src/styles/global.css` ‚Äî dark theme using Tailwind CSS v4 `@theme` block with custom colors
- **React**: configured via `@astrojs/react` integration; TSX components use `react-jsx` transform
- **Tailwind**: uses Vite plugin (`@tailwindcss/vite`) instead of PostCSS
- **TypeScript**: strict mode via `astro/tsconfigs/strict`; path aliases `@db/*`, `@lib/*`, and `@components/*`
- **Package manager**: pnpm (monorepo with `packages/widget`)

## Database

- **ORM**: Drizzle ORM with **PostgreSQL** (e.g. Supabase)
- **Schema**: `src/db/schema.ts` ‚Äî all tables and relations (pg)
- **Connection**: `src/db/index.ts` ‚Äî uses `DATABASE_URL` (Postgres), runs migrations on startup
- **Migrations**: `drizzle/` directory, generated by `drizzle-kit` (dialect: postgresql)
- **Config**: `drizzle.config.ts` ‚Äî set `DATABASE_URL` in env for generate/migrate/studio

## Auth

- Cookie-based sessions (`bugbash_token` cookie, httpOnly, 30-day expiry)
- No passwords ‚Äî email-only login for returning users
- First-user bootstrap via `/setup` (creates admin + org + default squads)
- Invite links via `/convite/[code]` for new users (admin/facilitator can generate)
- `src/lib/auth.ts` ‚Äî session create/validate/delete, email login, domain allowlist
- `src/lib/auth/invites.ts` ‚Äî invite code generation, verification, usage tracking
- `src/middleware.ts` ‚Äî protects routes; zero-user ‚Üí `/setup`, no session ‚Üí `/entrar`
- Login: POST `/api/auth/login` (email-only); Logout: POST `/api/auth/logout`
- No external identity provider ‚Äî email-only auth

## Key Technical Decisions

- **Postgres** (Supabase or any Postgres) for easy deploy; no file-based DB
- `vite.ssr.external: ['pg', 'pg-native', '@supabase/supabase-js', '@anthropic-ai/sdk', '@linear/sdk']` to avoid bundling native/heavy addons
- Auto-migration on startup (idempotent); for production, run `pnpm db:migrate` before starting the app
- Dark mode always on (`color-scheme: dark`)
- Language: `pt-BR` for UI labels
